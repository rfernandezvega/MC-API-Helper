Documentación y Guía de Uso

  1. Paso 1: Configurar la Aplicación en Marketing Cloud
    • Se deberá crear siempre la app en la BU PADRE y dar permisos a las BUs hijas y usuarios que vayan a poder tener acceso.
    • Ve a Setup → Apps → Installed Packages.
    • Crea un nuevo paquete de tipo Web App llamado MC API Helper.
    • El **Redirect URI** debe ser exactamente: https://127.0.0.1:8443/callback.
    • En el componente de API, asigna estos permisos:
      • Email: Read, Write
      • Automations: Read, Write, Execute
      • Journeys: Read, Write, Activate/Stop/Pause/Resume/Send/Schedule, Delete
      • Audiences: Read, Write
      • List and Subscribers: Read, Write
      • Data Extensions: Read, Write
      • Documents and Images: Read
      • Saved Content: Read
    • En el usuario que va a conectarse, en Permissions: Email > Content > Email > View
      • IMPORTANTE: La aplicación en Marketing Cloud puede tardar hasta 10 minutos en estar completamente operativa después de guardarla.
  2. Paso 2: Configurar un Cliente en esta App
    • Ve a la vista de Configuración de APIs.
    • Introduce un nombre para el cliente (ej: "Cliente X - Producción").
    • Rellena los datos (MID, Auth URI, Client ID, etc.) obtenidos del paso anterior.
    • Pulsa "Login" para guardar e iniciar la sesión. Se abrirá una ventana de Salesforce para que te autentiques.
    • En la parte inferior se pueden configurar las tablas sobre las que lanzar la búsqueda de clientes. Se debe indicar el nombre de la tabla, su External Key y el nombre del campo por el que buscar al cliente
    • Pulsa "Guardar". La configuración quedará almacenada en tu equipo. Esto principalmente es para la parte de las tablas.
  3. Log de Eventos
    • El panel lateral derecho se puede colapsar.
    • El bloque Mensajes muestra los procesos ejecutados y los posibles errores que se hayan podido dar al realizar una acción.
    • El bloque Llamada API, muestra todas las llamadas que se han hecho al realizar una acción con el endpoint y el payload.
    • El bloque Respuesta APIRespuesta API, muestra todas las respuestas que se han recibido al realizar una acción.
  1. Creación de Data Extensions

    Este flujo te permite crear una Data Extensions.

    1. Configuración:
      • Ve a Data Extensions → Creación de Data Extension para definir el Nombre, la Descripción y si es "Sendable".
      • Puedes incluir el identificador de una carpeta para que se cree donde quieras. Si no se indica, se crea en la carpeta Data Extensions.
      • Ve a Data Extensions → Configuración de Campos para añadir los campos.
      • Puedes hacerlo con:
        • Añadir Fila: Añade filas para configurar campos manualmente. Puedes elminar la fila seleccionandola y pulsando en el botón que aparece a la derecha.
        • Campos de Ejemplo: Mete un campo de cada tipo
        • Importar: Permite pegar desde el portapapeles, indicando que separador utilizar.
      • Finalmente, pulsa "Crear Data Extension" en la primera pantalla.
  2. Modificar o Añadir Campos a una DE Existente

    Este flujo te permite crear o actualizar campos en una Data Extension.

    1. Configuración:
      • Ve a Data Extensions → Recuperación y Borrado de Campos.
      • Introduce la External Key de la DE y pulsa "Recuperar Campos".
      • La estructura se cargará en la vista de Configuración de Campos. Añade las filas nuevas que necesites.
      • Pulsa "Crear/Actualizar Campos" para aplicar los cambios (esta acción no borra campos).
  3. Borrar campos a una DE Existente

    Este flujo te permite borrar un campo en una Data Extension. Esto no se puede hacer por pantalla si está en un grupo de atributos, pero sí por API

    1. Configuración:
      • En la misma vista de Recuperación y Borrado de Campos, introduce la External Key de la DE y pulsa "Recuperar Campos"
      • La estructura se cargará en la vista de Configuración de Campos y en la picklist.
      • Selecciona un campo en la picklist y pulsa "Borrar Campo" para eliminarlo.
  4. Documentar Data Extensions

    Este flujo te permite generar un csv con las Data Extensions de una carpeta. Util cuando hay que reivsar una nueva instancia

    1. Configuración:
      • En la misma vista de Recuperación y Borrado de Campos, introduce el ID numérico de una carpeta.
      • Pulsa "Documentar Data Extensions". La aplicación generará y descargará un fichero CSV con todas las DEs de esa carpeta y la estructura completa de sus campos.
  5. Calendario de Automatizaciones

    Una vista para visualizar los automatismos programados o los automatismos programados que lanzan journeys.

    • Refrescar Automatismos: Pulsa este botón para cargar todas las automatizaciones programadas en el calendario.
    • Refrescar Automatismos de Journeys: Pulsa este botón para cargar únicamente las automatizaciones que contienen una actividad de Journey.
    • Navegación: Haz clic en un día del calendario que esté resaltado. La lista de la derecha se actualizará con las ejecuciones de ese día. Si haces clic en el título de esa lista "Automatizaciones del Día", serás redirigido a la vista de "Gestión de Automatismos", pre-filtrada para mostrar solo las de ese día.
  6. Gestión de Automatismos

    Panel de control para visualizar, filtrar y ejecutar acciones sobre múltiples automatismos a la vez.

    • Al entrar por primera vez, se recuperarán y cargarán todos los automatismos de la instancia
    • Es posible filtrar por Nombre y Estado de los automatismos
    • La tabla permite seleccionar más de un automatismo. Volviendo a clicar, se deselecciona.
    • Lanzar acciones. Si se selecciona más de un automatismo, solo se pueden lanzar acciones si todos los automatismos seleccionados tienen el mismo estado.
      • Refrescar: Refresca el listado completo de automatismos. Esto se hace automáticamente al realizar cualquier acción.
      • Activar: Pone en marcha la programación de los automatismos seleccionados. Solo se habilita para automatismos en estado "PausedSchedule" o "Stopped".
      • Ejecutar: Lanza una ejecución única e inmediata de los automatismos seleccionados. Solo se habilita para automatismos en estado "PausedSchedule" o "Stopped".
      • Parar: Detiene la programación de los automatismos seleccionados. Solo se habilita para automatismos en estado "Scheduled" o "Ready".
  7. Clonador de Automatismo

    Una herramienta para clonar un automatismo y sus actividades.

    • Ve a Gestión de Automatismos.
    • Selecciona el automatismo que quieres clonar.
    • Se mostrará una nueva vista donde se puede:
      • Indicar el nombre del automatismo clonado. Por defecto se clona con _Copy
      • Seleccionar qué actividades se quieren clonar. Por ahora solo se clonan las Queries para las que se puede indicar el nombre de la Query copiada y la de la Data Extension
    • Todos los elementos se guardan en la misma carpeta donde estaban los originales.
    • Despues de seleccionar qué elementos clonar, pulsa en el botón de Clonar para ejecutar la clonación.
  8. Gestión de Journeys

    Panel de control para visualizar, filtrar y ejecutar acciones sobre múltiples journeys a la vez.

    • Al entrar por primera vez, se recuperarán y cargarán todos los journeys de la instancia
    • Es posible filtrar por Nombre, Tipo, Estado y Data Extension de los automatismos
    • La tabla muestra la información principal del Journey, donde se encuentra y que Data Extension se encuentra en su entrada.
    • Lanzar acciones. En funcion de la acción que se quiera realizar, se podrá seleccionar uno o varios journeys.
      • Refrescar: Refresca el listado completo de automatismos. Esto se hace automáticamente al realizar cualquier acción.
      • Comunicaciones: Recupera todas las comunicaciones de los Journeys seleccionados. Lo que se muestra es el nombre de la actividad en el Journey por lo que si se ha cambiado, no coincidirá con el contenido.
      • Dibujar: Permite crear una versión de texto del Journey de cara a la documentación. Es necesario haber recuperado las comunicaciones previamente, lo cual marca el Journey como Descargado
      • Copiar: Permite clonar un Journey, con toda su estructura y clona la Data Extension de entrada en la misma carpeta donde se encontrará la anterior. Por ahora solo para los de tipo "EmailAudience".
      • Parar: Permite parar varios Journeys a la vez
      • Borrar: Permite borrar varios Journeys a la vez, si son de Subtipo Quicksend
  9. Buscador de Data Extensions

    Panel de control buscar rutas de Data Extensions.

    • Selecciona por qué quieres buscar: Nombre o External Key
    • No es necesario indicar el nombre completo, se mostrarán las rutas de todas las Data Extension que contengan el texto indicado
    • Al pulsar en Buscar, se mostrarán las rutas en la tabla inferior
  10. Buscador de Origen de Datos

    Panel de control para buscar qué actividades rellenan los datos de una Data Extension (Queries e Imports).

    • Indica el nombre exacto de la Data Extension y pulsa en Buscar Orígenes.
    • Se mostrarán las actividades, el tipo y, si se encuentran dentro de una automatismo, en qué paso.
  11. Buscador de Clientes

    Panel de control para buscar clientes y su información relacionada.

    • Pon el valor por el que quieres buscar. Puede ser un identificador o el mail.
    • Pulsa en Buscar Cliente para empezar la búsqueda.
    • Se realizará una búsqueda por Subscriber y si no se encuentra se buscara por Contacto.
    • Si se ha encontrado un suscriptor, los botones DEs y Journeys estarán activos.
    • Si se ha encontrado un contacto, el botón Journeys estará activo.
      • DEs: Buscará lo que se haya indicado en el formulario en todas las DEs configuradas a nivel de Configuración de APIs del cliente al que se ha conectado. Se muestran todos los campos de la tabla
      • Journeys: Buscará lo que se haya indicado en el formulario en todos los Journeys. Si se encuentra en ese momento en alguno, lo mostrará en un tabla
  12. Buscador de Texto en Queries

    Panel de control para buscar Queries que contengan el texto indicado en el buscado.

    • Pon el valor por el que quieres buscar. Puede ser un identificador o el mail.
    • Pulsa en Buscar en Queries para empezar la búsqueda.
    • La tabla muestra el nombre de las queries encontradas y en qué automatismo y paso se encuentran, si es que está en alguno.
    • La columna de Query Text se puede ocultar/mostrar con la casilla del formulario.
  13. Clonador Masivo de Queries

    Esta herramienta te permite clonar múltiples Query Activities y sus Data Extensions de destino entre carpetas.

    1. Paso 1: Configuración de Carpetas
      • Ve a Funcionalidades → Clonador de Queries.
      • Introduce el nombre (o parte del nombre) de tres carpetas: la de origen de las Queries, la de destino de las Queries clonadas y la de destino de las DEs clonadas.
      • Pulsa "Buscar Carpetas". Se mostrarán tablas con las posibles rutas encontradas.
      • Selecciona una ruta de cada una de las tres tablas. El botón "Continuar" se habilitará.
    2. Paso 2: Selección y Clonación
      • Verás una lista de todas las Queries de la carpeta de origen.
      • Usa las casillas de verificación para seleccionar las que quieres clonar.
      • Puedes modificar el nombre de las DEs y Queries que se van a crear. Si no, se crearán añadiendo el sufijo `_Copy.
      • Pulsa "Clonar Seleccionadas". La aplicación clonará cada query y su DE de destino a las carpetas que elegiste.
  14. Validador de Email

    Una herramienta sencilla para verificar la validez de una dirección de correo con los validadores que utiliza Marketing Cloud.

    • Ve a Funcionalidades → Validador de Email.
    • Introduce la dirección de correo y pulsa "Validar Email".
    • El resultado te indicará si el email es válido según las validaciones de sintaxis, registros MX y List Detective de Marketing Cloud.
  15. Gestión de Cloud Pages

    Panel de control para visualizar y filtrar Cloud Pages.

    • Ve a Funcionalidades → Gestión de Cloud Pages.
    • Se cargará la tabla con todas las Cloud Pages de la instancia.
    • Importante: no es posible recuperar el identificador real (el que se utiliza en CloudPageURL()) porque ninguna API lo devuelve. 🤷🏻
  • Error de autenticación

    Si se ha creado la aplicación en MC, se ha configurado el usuario en Marketing Cloud API Helper y luego se cambian los permisos en la aplicación de MC, lo mejor es deslogearse y volverse a logear para que recupere el token con los permisos correctos.

  • La DE se crea en la carpeta raíz

    Para crear una DE en una carpeta específica, debes usar su ID numérico, no su nombre. Lo mismo ocurre para documentar las Data Extensions de una carpeta

  • "Crear/Actualizar campos" es un "upsert"

    Esta acción actualiza campos que ya existen o crea los que son nuevos, sin borrar los que no están en la tabla.

  • Longitud en campos Decimales

    Para campos de tipo Decimal, usa el formato "total_dígitos,número_decimales", por ejemplo, 18,2.
