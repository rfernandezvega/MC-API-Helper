Documentación y Guía de Uso

	

		
		

			
			
			
			
		

		
		

			
    
				
				
  1. 
					
    
						
							
							Paso 1: Configurar la Aplicación en Marketing Cloud
						
					
    
					
    
						
    
							
      
								
    • Se deberá crear siempre la app en la BU PADRE y dar permisos a los usuarios. Los permisos se dan en el propio paquete que se ha creado en la pestaña ACCESS
								
      • 
								
    • Si se quiere usar en otras BUs, se debe dar permiso a cada BU y establecer una
									conexción por cada BU.
      • 
								
    • Ve a Setup → Apps → Installed Packages. 
      • 
								
    • Crea un paquete de tipo Web App llamado MC API Helper. 
      • 
								
    • En Redirect URI pon lo siguiente:
									https://127.0.0.1:8443/callback. 
      • 
								
    • Pon los siguientes permisos:
									
        
										
      • Email: Read, Write
        • 
										
      • Automations: Read, Write, Execute
        • 
										
      • Journeys: Read, Write, Activate/Stop/Pause/Resume/Send/Schedule,
											Delete
        • 
										
      • Audiences: Read, Write
        • 
										
      • List and Subscribers: Read, Write
        • 
										
      • Data Extensions: Read, Write
        • 
										
      • Documents and Images: Read
        • 
										
      • Saved Content: Read
        • 
										
      • File Locations: Read
        • 
										
      • Users: Read
        • 
									
      
								
      • 
							
    
						
    
					
    
				
    2. 
				
				
  2. 
					
    
						
							
							Paso 2: Configurar un Cliente en esta App
						
					
    
					
    
						
    
							
      
								
    • Ve a Configuración de APIs e introduce los datos del paquete. 
      • 
								
    • Pulsa "Login" para autenticarte vía browser. 
      • 
								
    • Se levantará una ventana con el login de Marketing Cloud. Pon tus credenciales
      • 
								
    • Si da algún problema, vuelve a probarlo porque puntualmente MC puede no gestionar bien la conexión. Si sigue fallando, revisa que el usuario tiene permiso en la app del Paso 1
      •

Data Extensions

					
    
						
  • 
							
    
								Creación de Data Extensions
							
    
							
    
								
    
									
    Esta vista está ligada a la de Configuración de Campos y permite establecer la configuración de la Data Extension que se quiere crear.
    
									
    Aquí se debe establecer el Nombre, Descripción (obligatorio) y si es "Sendable". Además, se puede realizar una búsqueda de carpetas para indicar dónde se debe guardar
    
									
    Se puede indicar si es una Data Extension Sendable marcando la casilla correspondiente. Para ello, es necesario haber configurado los campos en Configuración de Campos
									ya que requiere seleccionar uno de ellos para establecer la relación con el suscriptor.
    
									
    
										
    
											
											Panel de configuración Base
										
    
										
    
											
											Selector de carpetas
										
    
										
    
											
											Panel de configuración Sendable
										
    	
									
    
									
    Para configurar los campos se debe ir a la vista Configuración de Campos.
    	
									
    En esta vista se puede:
										
      
											
    • Importar un documento con los campos, por ejemplo cuando los copiamos del modelo de datos de Salesforce. Para ello solo se debe pegar el listado de campos e indicar el tipo de separador.
      • 
											
    • Crear/Actualizar Campos, cuando se quieren añadir nuevos campos a una tabla existente o actualizar longitudes en campos
      • 
											
    • Crear campos de ejemplo, para ver cómo se crea un campo de cada tipología
      • 
											
    • Limpiar toda la tabla de campos. Esto siempre mantendrá una fila vacía
      • 
											
    • Añadir filas a la tabla
      • 
											
    • Eliminar filas. Para ello seleccionando una fila, aparecerá una x a la derecha de la fila para poder eliminarla
      • 
											
    • Ordenar filas. Seleccionando una fila y usando las flechas, se puede ordenar la tabla. Esto es interesante sobre todo cuando queremos crear una copia de 
												una tabla existente pero con un orden correcto, por ejemplo manteniendo los identificadores al principio.
												NO es posible ordenar campos sobre una Data Extension existente
											
      • 
										
    
									
    
									
    
										
    
											
											Importar campos
										
    	
										
    
											
											Gestionar campos
										
    	
										
    
											
											Selector de tipo de campo
										
    										
									
    																	
									
    Una vez configurado, pulsando en Crear Data Extension se creará la Data Extension. Importante, no se limpia la configuración una vez creada 
										por si se quieren seguir haciendo acciones sobre la misma o seguir creando Data Extensions parecidas

								Buscador de Clientes
							
    
							
    
								
    
									
    El buscador de Clientes pretende tener una forma rápida de buscar la información relacionada con un cliente de forma rápida, 
										por ejemplo, cuando se pregunta qué comunicaciones ha recibido, tracking relacionado, si está en una data extension de exclusión, etc. 
										Esto sin necesidad de tener que ir a cada tabla o lanzar queries por separado
    
									
    Además permite buscar en qué Journeys se encuentra en ese momento y permite expulsarlos de estos de forma masiva. Es decir, se puede seleccionar más de un 
										journey del que expulsarlo a la vez.
    
									
    Para usar la funcionalidad de búsqueda en tablas es necesario haber configurado las tablas donde buscar en 
										General > Configuración de APISs > DEs Búsquead de Clientes. Estas se podrán seleccionar para buscar y mostrará todo los campos de la misma, 
									filtrado por el campom de búsqueda.

Contenidos

					
    
						
  • 
							
    
								Contenidos
							
    
							
    
								
    
									
    
										Esta vista permite descargar todos los contenidos de la instancia para poder hacer búsquedas sobre ellos. 
										En este caso Marketing Cloud no pone una forma fácil de obtenerlo así que es necesario hacer algun paso 
										en el propio navegador para obtener lo contenidos.
									
    

									
    
										Para importarlos, pulsa en el botón Importar que mostrará una ventana con las instrucciones a seguir. 
										¡Importante! Si al acceder al enlace que se indica en las instrucciones ves un UNAUTHORIZED, 
										abre Content Builder y vuelve a intentarlo.
									
    

									
    
										Una vez pegado el código, pulsa Enter y espera a que se termine de ejecutar. Cuando termine, copia lo 
										que ha generado y pégalo donde indica la app. Puede que le cuesta copiarse porque el texto es enorme. De la 
										misma forma, al pegarlo en la app tardará asi qué NO LE DES A PEGAR VARIAS VECES.
									
    

									
    
										Una vez importados, los contenidos se guardarán asociados al cliente al que se está conectado, por lo que solo será 
										necesario volver a importar si se quiere buscar en contenidos nuevos. El buscador filtrará los contenidos 
										que contengan la cadena de texto. Aunque no se muestre, también busca en el cuerpo de los emails y se puede 
										utilizar por ejemplo para saber qué mails utilizan una plantilla (buscando por el nombre de la misma).
