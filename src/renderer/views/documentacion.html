Documentación

Configuración principal

  • Configurar un Cliente: Ve a Configuración de APIs. Introduce un nombre para tu proyecto o cliente, rellena los datos de la aplicación de Marketing Cloud (MID, Auth URI, Client ID, etc.) y pulsa "Guardar". La configuración se guardará localmente.
    • Almacenamiento Local: Toda la configuración de los clientes se guarda en el almacenamiento local del navegador. No se envía a ningún servidor externo. Si borras la caché de tu navegador o cambias de equipo, perderás esta información.
    • Recuperar Token: (Opcional) Puedes pulsar "Recuperar Token" en el menú superior de esta vista para comprobar que las credenciales son correctas. La aplicación lo hará automáticamente cuando ejecutes otras funciones.
  • Clientes: Puedes guardar varios clientes y cambiar entre ellos ya sea desde la lista de Configuración de APIs o en el panel izquierdo de Clientes.

Data Extensions

Configuración de Data Extension

  • Configurar la Data Extension: Ve a Configuración de Data Extension. Define el nombre, descripción y carpeta. La external Key se generará automáticamente.
  • Vincular Subscriber Key: Si la DE debe ser "Sendable", selecciona la casilla para habilitar la lista de selección de campo.
    • Campos: La lista de selección para vincular el Subscriber Key muestra automáticamente los campos disponibles en la vista de Campos.
  • Creación: Pulsa en el botón Crear Data Extension para que se cree en la carpeta indicada. Revisa el panel derecho de Logs por si se ha devuelto algún tipo de error.

Campos

  • Esta vista muestra una tabla para indicar los campos que deberá tener la Data Extension.
  • Añadir Campos: Para que la tabla muestre campos hay varias formas.
    • Usa "Añadir Fila" para añadir filas manualmente y poder configurar los campos. Los tipos de campo son:
      • Text
      • Boolean
      • Date
      • Decimal
      • Phone
      • EmailAddress
      • Locale
      • Number
    • Usa "Crear Campos Dummy" para generar un ejemplo rápido de un campo por cada tipo.
    • Usa "Importar..." para pegar una lista de campos desde Excel o un CSV. Podrás seleccionar el separador que quieras.
  • Usa "Limpiar Campos" para eliminar todos los campos de la tabla.
  • Usa "Crear campos" para crear los campos de la tabla en la Data Extension definida en Gestión de Campos.
  • Define qué campos son Primary Key (PK) u obligatorios.
  • Seleccionando una fila, podrás:
    • Ordenar: Con los botones Subir y Bajar podrás organizar los campos. Esto es útil cuando tenemos una tabla que queremos dejar bien ordenada. Con la funcionalidad de recuperar tablas, podremos descargarla, ordenar los campos y crearla de nuevo.
    • Borrar: A la derecha de la fila seleccionada aparecerá un botón de borrado para eliminar la fila.

Gestión de Campos

  • Esta vista muestra un campo para indicar la DE sobre la que se quiere trabajar para eliminar campos o para crearlos.
  • Usa "Recuperar campos" para recuperar todos los campos de la tabla. Aparecerán en Campos y en la lista Campo a eliminar de esta vista.
  • Usa "Borrar campo" para borrar el campo seleccionado en la lista. Esto es especialmente útil para eliminar campos que se utilizan en Data Extensions conectadas en un grupo de atributos, ya que por interfaz no se puede hacer.

Funcionalidades

Calendario

  • Pulsa en Refrescar datos para recuperar los automatismos que tienen actividades de lanzamiento de Journeys. Tardará un rato.
  • Si hay automatismos programados, aparecerán en la fecha correspondiente en el calendario.
  • Pulsando en la fecha, aparecerán los automatismos en el bloque de Automatizaciones del día.

Búsqueda de Data Extension

  • Selecciona por qué campo quieres realizar la búsqueda: Nombre o Clave externa
  • Rellenar el Nombre o la Clave externa y pulsa en Buscar
  • Si la encuentra, aparecerá la ruta en el panel de Resultados.

Validador de email

  • Rellena el mail que quieres validar y pulsa en Validar Email
  • En el panel de Resultado de la Validación se indicará si es válido o no para Marketing Cloud.

Buscador de origenes

  • Esta funcionalidad busca qué actividades rellenan información en la Data Extension indicada. Importante: se busca por Nombre
  • En el panel de Actividades Encontradas se mostrarán las actividades (Queries o Imports) que la rellenan.
  • Error de autenticación: Revisa que el MID, Client ID y Client Secret sean correctos. Asegúrate de que la aplicación API creada en Marketing Cloud es de tipo "Server-to-Server".
  • La DE se crea en la carpeta raíz: Si quieres crear la DE en una carpeta específica, debes proporcionar el ID de la Carpeta (un número), no su nombre. Puedes encontrarlo en la URL al navegar a esa carpeta en Marketing Cloud.
  • DE Sendable sin Subscriber Key: Si marcas una DE como "Sendable", es obligatorio seleccionar un campo que actuará como Subscriber Key y que este campo esté marcado como "Obligatorio" en la tabla de campos.
  • Error "DE ya existe": Si intentas crear una DE con una External Key que ya existe en la Business Unit, la API devolverá un error.
  • Acceso a Business Units Hijas: La aplicación API debe crearse en la BU padre (Enterprise) y se le deben dar permisos explícitos para acceder a las BUs hijas que necesites.
  • Error "Bad Request" con enlaces: Si pegas una URL en un campo de texto que contiene un "&", la API SOAP puede fallar. Reemplaza & por &amp; para solucionarlo.
  • Permisos de la Aplicación API: Para un funcionamiento completo, la aplicación API en Marketing Cloud necesita permisos de Lectura y Escritura para Data Extensions (Data > Data Extensions > Read & Write), de Lectura para Automatizaciones (Automations > Read) y de Lectura para Email (Email > Read).
  • Creación vs. Actualización de Campos: La macro "Crear campos" es en realidad un "upsert". Esto significa que si un campo con el mismo nombre ya existe, actualizará sus propiedades. Si no existe, lo creará.
  • Tipos de Campo: Asegúrate de que los tipos de campo coinciden con los soportados por Marketing Cloud (Text, Number, Date, Boolean, EmailAddress, Phone, Locale, Decimal).
  • Longitud en Decimales: Para campos de tipo "Decimal", la longitud debe especificarse como "total,decimales". Por ejemplo, 18,2 para un campo que admite hasta 18 dígitos, con 2 de ellos siendo decimales.
