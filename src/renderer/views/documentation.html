Documentación y Guía de Uso

  1. Paso 1: Configurar la Aplicación en Marketing Cloud
    • Se deberá crear siempre la app en la BU PADRE y dar permisos a los usuarios.
    • Si se quiere usar en otras BUs, se debe dar permiso a cada BU y establecer una conexción por cada BU.
    • Ve a Setup → Apps → Installed Packages.
    • Crea un paquete de tipo Web App llamado MC API Helper.
    • En Redirect URI pon lo siguiente: https://127.0.0.1:8443/callback.
    • Pon los siguientes permisos:
      • Email: Read, Write
      • Automations: Read, Write, Execute
      • Journeys: Read, Write, Activate/Stop/Pause/Resume/Send/Schedule, Delete
      • Audiences: Read, Write
      • List and Subscribers: Read, Write
      • Data Extensions: Read, Write
      • Documents and Images: Read
      • Saved Content: Read
  2. Paso 2: Configurar un Cliente en esta App
    • Ve a Configuración de APIs e introduce los datos del paquete.
    • Pulsa "Login" para autenticarte vía browser.
    • Se levantará una ventana con el login de Marketing Cloud. Pon tus credenciales
    • Si da algún problema, vuelve a probarlo porque puntualmente MC puede no gestionar bien la conexión. Si sigue fallando, revisa que el usuario tiene permiso en la app del Paso 1

General

  • Descarga de logs

    La aplicación dispone de un panel lateral derecho donde se van registrando todas las acciones y llamadas que realiza la app. Pulsa en los iconos < o > para mostrar u ocultar el panel.

    Mostrar/Ocultar panel de log

    Pulsando en Log de Eventos se abrirá la ventana para descargar los logs con la fecha. Esto es útil por si os da algún error raro, para poder revisar que llamadas se han hecho

    Descarga de logs
  • Configuración de APIs > DEs Búsqueda de Clientes

    Además de los pasos indicados en Primeros Pasos, dentro de la vista de Configuración de APIs está la pestaña DEs Búsqueda de Clientes.

    Esta permite establecer una serie de Data Extension que permitirán tener una visión del cliente dentro de la vista Funcionalidades > Buscadores > Clientes Es útil cuando nos piden saber qué comunicaciones ha recibido un cliente, qué estado tiene en alguna tabla, logs, etc, sin necesidad de lanzar múltiples queries.

    Para configurarlo, es necesario indicar:

    • Nombre de la Data Extension donde buscar
    • External Key de la Data Extension donde buscar
    • Por qué campo se debe buscar al cliente (el que coincida con el id)

    Panel de DEs Búsqueda de Clientes
    Exportación de fichero con nombre del cliente

    Si se ha configurado esta parte antes de darle al login, se guardarán las tablas automáticamente.

    Si se añaden tablas posteriormente, es necesario darle al botón de guardar de la parte superior.

    Una vez configuradas, se pueden exportar para que otra persona del equipo no tenga que configurarlas manualmente

Data Extensions

  • Creación de Data Extensions

    Esta vista está ligada a la de Configuración de Campos y permite establecer la configuración de la Data Extension que se quiere crear.

    Aquí se debe establecer el Nombre, Descripción (obligatorio) y si es "Sendable". Además, se puede realizar una búsqueda de carpetas para indicar dónde se debe guardar

    Se puede indicar si es una Data Extension Sendable marcando la casilla correspondiente. Para ello, es necesario haber configurado los campos en Configuración de Campos ya que requiere seleccionar uno de ellos para establecer la relación con el suscriptor.

    Panel de configuración Base
    Selector de carpetas
    Panel de configuración Sendable

    Para configurar los campos se debe ir a la vista Configuración de Campos.

    En esta vista se puede:

    • Importar un documento con los campos, por ejemplo cuando los copiamos del modelo de datos de Salesforce. Para ello solo se debe pegar el listado de campos e indicar el tipo de separador.
    • Crear/Actualizar Campos, cuando se quieren añadir nuevos campos a una tabla existente o actualizar longitudes en campos
    • Crear campos de ejemplo, para ver cómo se crea un campo de cada tipología
    • Limpiar toda la tabla de campos. Esto siempre mantendrá una fila vacía
    • Añadir filas a la tabla
    • Eliminar filas. Para ello seleccionando una fila, aparecerá una x a la derecha de la fila para poder eliminarla
    • Ordenar filas. Seleccionando una fila y usando las flechas, se puede ordenar la tabla. Esto es interesante sobre todo cuando queremos crear una copia de una tabla existente pero con un orden correcto, por ejemplo manteniendo los identificadores al principio. NO es posible ordenar campos sobre una Data Extension existente

    Importar campos
    Gestionar campos
    Selector de tipo de campo

    Una vez configurado, pulsando en Crear Data Extension se creará la Data Extension. Importante, no se limpia la configuración una vez creada por si se quieren seguir haciendo acciones sobre la misma o seguir creando Data Extensions parecidas

    Creación Data Extension OK
  • Recuperar y borrar campos de una Data Extension

    Esta funcionalidad es útil para copiar una Data Extension o eliminar campos que no se pueden eliminar por pantalla

    Indica la clave externa de la tabla y pulsa Recuperar Campos. Los campos recuperados aparecerán en la lista y en la pestaña de Configuración de Campos

    Para borrar un campo, solo se debe seleccionar en la lista y pulsar en Borrar Campo

    Recuperacion de campos
    Borrado de campos
  • Documentar Data Extensions

    Esta funcionalidad permite generar un fichero CSV con la estructura completa de todas las DEs contenidas en una carpeta específica.

    ¿Para qué es útil? Por ejemplo cuando queremos tener identificados que campos de Salesforce estan integrados. Recuperando la carpeta de Sync DEs podemos documentarlo y luego compararlo con el modelo de datos de SF

    Para documentar, selecciona la carpeta y pulsa en Documentar DEs

    Documentar DEs

Funcionalidades

  • Buscadores Avanzados
    • Buscador de DEs: Encuentra la ruta de una DE por Nombre/Key.
    • Origen de Datos: Qué Query o Import alimenta a una DE.
    • Buscador de Clientes: Localiza suscriptores en DEs configuradas y Journeys.
    • Texto en Queries: Busca fragmentos de código SQL en toda la cuenta.
  • Calendario de Automatizaciones

    Visualiza gráficamente qué automatismos están programados para hoy o cualquier fecha seleccionada.

  • Gestión de Automatismos

    Panel masivo para Activar, Parar o Ejecutar una vez múltiples automatismos. Incluye herramienta de clonación completa.

  • Gestión de Journeys

    Clonación de Journeys, parada masiva y generación de documentación visual (dibujo) de la lógica del Journey.

  • Clonador Masivo de Queries

    Herramienta para mover/duplicar múltiples Query Activities y sus DEs de destino entre diferentes carpetas.

  • Creación de Carpetas

    Crea árboles de carpetas pegando un esquema de texto tabulado.

  • Validador de Email

    Verifica direcciones contra List Detective y registros MX de Marketing Cloud.

No se han encontrado funcionalidades que coincidan con la búsqueda.
  • ⚠️ Error de autenticación

    Si cambias permisos en MC, es necesario desloguearse y volver a loguearse en la app para refrescar el token.

  • ⚠️ Carpetas no encontradas

    Para crear o documentar DEs, usa siempre el buscador de carpetas para obtener el ID numérico correcto.

  • 💡 Upsert de campos

    La acción "Actualizar campos" no borra nada; solo añade lo nuevo o modifica lo existente.

  • 💡 Decimales

    En el campo longitud para decimales usa la coma: 18,2.
